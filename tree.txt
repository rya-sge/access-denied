PlantUML source registry
========================

This file maps each article (in _posts/) to the PlantUML (.puml) source files
that produced its diagrams, and where those sources live under assets/article/.

Why this file exists:
- PlantUML diagram SOURCE (.puml) is NEVER embedded in an article anymore.
  Only the rendered PNG is embedded. The mindmap PNG appears in the article's
  Conclusion/Summary and in the frontmatter `image:` field; supporting concept
  and workflow PNGs (when present) are kept as standalone assets only.
- Because the .puml source is no longer in the .md file, this registry is the
  way to find the original source when an article is later modified, so the
  diagram can be edited and re-rendered instead of reverse-engineered from PNG.

Convention:
- Each article has exactly one mindmap .puml (tag: mindmap).
- It may also have supporting diagrams (tags: concept, workflow, ...).
- Mindmap source is usually named after the article or with a -mindmap suffix.
- Supporting diagrams use descriptive kebab-case names (e.g. *-concept.puml).
- Each .puml has a sibling .png produced by: plantuml -tpng <file>.puml
